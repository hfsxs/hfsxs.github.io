Prometheus监控告警数据持久化

Prometheus监控系统的告警信息基于告警规则生成，并将生成的告警信息存储为时间序列，再发送给Alertmanager进行转发，整个过程一直专注于实时化告警的处理，并不会持久化存储。这一设计虽然简化了系统设计的复杂性，但大量的告警无疑将会影响故障排查和响应效率，也难以直观地区分告警的轻重缓急。此外，也不能对历史告警进行检索与统计，无法对业务系统的瓶颈与缺陷进行分析。Alertsnitch工具弥补了这一不足，其工作机制是将Alertmanager的告警信息捕获，并将其写入MySQL或PostgreSQL关系数据库，以便进行后续的审计或分析‌

1.配置Go语言环境

1.1 安装Go语言程序

wget https://golang.google.cn/dl/go1.24.0.linux-amd64.tar.gz
tar -xzvf go1.24.0.linux-amd64.tar.gz && mv go /usr/local

1.2 配置环境变量

sudo vi /etc/profile


export GO_HOME=/usr/local/go
export PATH=$PATH:$GO_HOME/bin
export GOPATH=$GO_HOME/tagert

1.3 生效环境变量

source /etc/profile

1.4 配置国内代理

go env -w GOPROXY=https://goproxy.cn

2.安装Alertsnitch

2.1 下载Alertsnitch安装包

https://github.com/yakshaving-art/alertsnitch

2.2 编译Alertsnitch

tar -xzvf alertsnitch-0.2.1.tar.gz && cd alertsnitch-0.2.1
sudo go install && ll /usr/local/go/tagert/bin
sudo mv /usr/local/go/tagert/bin/alertsnitch

2.3 创建启动脚本

sudo vi /lib/systemd/system/alertsnitch.service


[Unit]
Description=Alertsnitch
Documentation=https://github.com/yakshaving-art/alertsnitch
After=network.target

[Service]
Type=simple
User=root
Group=root
Environment="ALERTSNITCH_DSN=root:123456@tcp(10.1.11.188:3306)/alertsnitch"
ExecStart=/usr/local/bin/alertsnitch
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

3.创建数据库

3.1 修改数据库初始化脚本

cd alertsnitch-0.2.1/script.d
vi bootstrap_mysql.sh 
#!/bin/bash

set -EeufCo pipefail
IFS=$'\t\n'

echo "Creating DB"
mysql --user=root --password="123456" -e "CREATE DATABASE IF NOT EXISTS alertsnitch;"

echo "Creating bootstrapped model"
mysql --user=root --password="123456" "alertsnitch" < ../db.d/mysql/0.0.1-bootstrap.sql

echo "Applying fingerprint model update"
mysql --user=root --password="123456" "alertsnitch" < ../db.d/mysql/0.1.0-fingerprint.sql

echo "Done creating model"

3.2 初始化数据库

sh bootstrap_mysql.sh

4.启动Alertsnitch

sudo systemctl daemon-reload
sudo systemctl start alertsnitch.service
sudo systemctl enable alertsnitch.service

5.配置Alertmanager

5.1 配置Alertsnitch关联Alertmanager

sudo vi /usr/local/prometheus/alertmanager.yml 


route:
  group_by: ['alertname','cluster']
  # 设置组内告警发送的等待时间，即组内收到第一个告警后的发送等待时间，目标是等待组内新增的告警以便同时合并发送，默认为30s
  group_wait: 10s
  # 设置组内不同批次告警发送的时间间隔，默认为5m
  group_interval: 10s
  # 设置告警未解决时重复发送的时间间隔，且此期间组内无新增告警，默认4h，建议根据告警严重程度进行设置
  repeat_interval: 1h 
  # 设置默认告警接收者，即未被子路由的receivers.name选项匹配到的告警接收者
  receiver: 'wechat'
  routes:
  - receiver: 'wechat'
    continue: true
  - receiver: 'alertsnitch'
    continue: true

receivers:
  - name: 'wechat'
    webhook_configs:
    - url: 'http://10.1.11.240:9095/qywechat'
      send_resolved: true
  - name: 'alertsnitch'
    webhook_configs:
      - url: 'http://10.1.11.182:9567/webhook'
        send_resolved: true

5.2 重启Alertmanager

sudo systemctl restart alertmanager.service
