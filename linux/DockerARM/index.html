Docker构建跨架构镜像

云计算时代，由于功耗低、高性能及指令集的优势，越来越多的云厂商开始选择基于ARM体系进行构建云服务。AWS率先发布的Graviton 2，到Apple的M1芯片，再到华为的鲲鹏体系，ARM体系逐渐成为了未来的发展趋势与方向，并据此引发了大量的适配测试需求。Docker的Manifest特性即为此而生，其实质上是一个用于存放不同OS/Arch的镜像信息清单列表，支持用户在不同系统架构的机器上分别运行不同架构的镜像

1.Docker开启Manifest

1.1 开启实验功能

sudo mkdir /root/.docker
sudo vi /root/.docker/config.json


{
  "auth": {},
  "experimental": "enabled"   
}

1.2 开启Manifest特性

sudo vi /etc/docker/daemon.json


{
  "registry-mirrors": ["https://registry.docker-cn.com"],
  "experimental": true  
}

1.3 重启Docker

sudo systemctl restart docker.service

2.拉取ARM架构镜像

sudo docker pull --platform=arm64 nginx:alpine

3.安装QEMU模拟器

sudo apt-get install -y qemu-user-static binfmt-support

4.注册QEMU到内核

sudo docker run --rm --privileged multiarch/qemu-user-static --reset -p yes

5.启动ARM镜像

sudo docker run --platform linux/arm64 -it -d -p 8080:80 nginx:arm
