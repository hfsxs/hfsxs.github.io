Docker镜像仓库管理

Repository，即镜像仓库，用于镜像的集中存储与管理。镜像仓库服务器存储多个仓库，每个仓库又包含了多个镜像，每个镜像有不同的标签。镜像仓库可理解为具体的项目或目录，如仓库地址hub.docker/nginx:v20.0，hub.docker是镜像仓库服务器地址，nginx是仓库名，v20.0是镜像版本号

Docker镜像仓库分为两类，即公共仓库和私有仓库。公共仓库，Public Registry，最大的公共仓库是Docker Hub，由docker官方维护，存放了数量庞大的镜像供用户下载，本地用户家目录的.docker/config.json文件存储用户的认证信息；私有仓库，Private Registry，本地服务器所建的镜像仓库，搭建方式有两种，即官方工具docker-registry和开源工具Harbor

1.registry

1.1 拉取镜像

sudo docker pull registry

1.2 创建并启动registry容器

sudo docker run -p 5000:5000 -d --restart=always \
-v /var/lib/docker/images:/var/lib/registry \
--name sword-registry registry

1.3 http方式测试镜像仓库

1.3.1 配置像拉取策略

sudo vi /etc/docker/daemon.json


{
 "insecure-registries":["172.16.100.100:5000"]
}

# 重启docker程序
systemctl restart docker

1.3.2 镜像推送私有仓库

# 更改nginx镜像标签
docker tag nginx:latest 172.16.100.100:5000/nginx:v1.0.0
# 推送到私有仓库
docker push 172.16.100.100:5000/nginx:v1.0.0

1.3.3 查看仓库镜像

# 显示所有镜像
curl http://172.16.100.100:5000/v2/_catalog
# 显示nginx镜像的标签
curl http://172.16.100.100:5000/v2/nginx/tags/list

1.3.4 拉取镜像

# 删除本地nginx镜像
docker rmi 172.16.100.100:5000/nginx:v1.0.0
# 从私有仓库拉取nginx镜像
docker pull 172.16.100.100:5000/nginx

1.4 配置Nginx

1.4.1 创建SSL自签名证书

sudo mkdir -p /etc/nginx/ssl && cd /etc/nginx/ssl
openssl req -x509 -newkey rsa:4096 -keyout reg.ops.com.key -out reg.ops.com.crt -days 36500 -nodes -subj "/CN=reg.ops.com"

1.4.2 创建配置文件

sudo vi /etc/nginx/conf.d/registry.conf


server {

    listen       443 ssl;
    server_name  localhost;

    charset utf-8;

    ssl_certificate      /etc/nginx/ssl/reg.ops.com.crt;
    ssl_certificate_key  /etc/nginx/ssl/reg.ops.com.key;

    ssl_session_cache    shared:SSL:1m;
    ssl_session_timeout  5m;

    ssl_ciphers  ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4:!DHE;
    ssl_prefer_server_ciphers  on;
    ssl_protocols TLSv1 TLSv1.1 TLSv1.2;

    location / {

        auth_basic "Nginx Server Auth";
        auth_basic_user_file /etc/nginx/conf.d/.auth_list;

        access_log  /var/log/nginx/registry_access.log  main;
        error_log  /var/log/nginx/registry_error.log;

        proxy_pass http://127.0.0.1:5000;

        }
}

1.4.3 操作系统信任SSL证书

# Debian/Ubuntu
sudo cp /etc/nginx/ssl/registry.crt /etc/ssl/certs
# CentOS
sudo cp /etc/nginx/ssl/registry.crt /etc/pki/tls/certs

1.4.4 重载Nginx

sudo nginx -t
sudo nginx -s reload

1.4.5 验证镜像仓库

sudo docker tag nginx reg.ops.com/nginx
sudo docker push reg.ops.com/nginx

2.Harbor

2.1 配置域名解析

sudo vi /etc/hosts


172.16.100.100	reg.ops.com

2.2 部署docker-compose

# 下载docker-compose
curl -L https://github.com/docker/compose/releases/download/1.25.0/docker-compose-`uname -s`-`uname -m` -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

2.3 创建SSL证书

sudo mkdir -p /data/ssl
openssl req -x509 -newkey rsa:4096 -keyout reg.ops.com.key -out reg.ops.com.crt -days 36500 -nodes -subj "/CN=reg.ops.com"

2.4 安装Harbor

cd /opt && wget https://github.com/goharbor/harbor/releases/download/v2.15.1/harbor-offline-installer-v2.0.0.tgz
tar -xzvf harbor-offline-installer-v2.0.0.tgz
cd harbor && cp harbor.yml.tmpl harbor.yml

2.5 创建配置文件

sudo vi harbor.yml



# 设置仓库域名
hostname: reg.ops.com
# 设置SSL证书路径
certificate: /data/ssl/reg.ops.com.crt
private_key: /data/ssl/reg.ops.com.key
# 设置仓库管理员admin登录密码
harbor_admin_password: Harbor12345
# 设置镜像数据目录
data_volume: /data
# 设置仓库日志目录
location: /var/log/harbor

2.6 安装Harbor

./install.sh

2.7 验证仓库

# 登录仓库
docker login reg.sword.com
# 设置镜像标签，带上项目名称
docker tag nginx reg.sword.com/library/nginx
docker push reg.sword.com/library/nginx
docker pull reg.sword.com/library/nginx

2.8 仓库启停

# 启动Harbor
docker-compose up -d

# 停止Harbor
docker-compose down
