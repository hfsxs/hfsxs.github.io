Hadoop集群部署

Hadoop，Apache基金会基于Java开发和维护的开源分布式计算与存储框架，核心功能是为庞大的计算机集群提供可靠的、统一且稳定的、可伸缩的存储和计算环境。Hadoop使用简单的编程模型跨计算机群集分布式处理大型数据集，其设计理念是将数据和计算任务分布到多个计算节点，通过分布式计算来提高处理效率，以解决单台机器处理大数据时的瓶颈问题

体系架构

核心部件包括分布式文件系统 (Hadoop DFS，HDFS) 和 MapReduce

应用场景

1.互联网行业

1.1 日志分析

  • 用户行为分析，互联网公司每天都会产生海量的用户访问日志，记录了用户的登录时间、浏览页面、停留时长、点击链接等信息。通过 Hadoop将这些日志数据收集存储在HDFS，再利用MapReduce或Spark等计算框架进行处理分析，以统计出用户的活跃度、最受欢迎的页面、用户在不同页面间的跳转路径等，为优化网站布局、提升用户体验、精准推送广告等提供依据，如电商网站根据用户行为分析来推荐符合其购买倾向的商品，提高商品转化率

  • 系统性能监测，服务器端会生成大量的系统日志，记录了CPU使用率、内存占用、网络带宽消耗、请求响应时间等指标。通过Hadoop对这些日志进行分析，能及时发现系统性能瓶颈，排查故障隐患，如判断哪个时间段服务器负载过高、哪些服务频繁出现超时等情况，以便运维人员采取相应措施优化服务器配置和网络架构

1.2 搜索引擎

百度、谷歌这类搜索引擎需要抓取海量网页并对其内容进行索引处理，Hadoop可助力完成网页数据的存储（HDFS）及索引构建过程中的大规模数据并行计算（MapReduce等），通过对网页文本内容进行词频统计、倒排索引构建等操作，使得用户在搜索时能够快速定位到相关网页，提高搜索结果的准确性和检索速度

2.金融行业

2.1 风险评估与信用分析

金融机构会收集客户的多维度数据，如个人基本信息、交易记录、借贷历史、消费行为等海量资料。利用 Hadoop 存储这些复杂多样的数据，并通过数据挖掘算法（基于 MapReduce 或 Spark 等框架实现）进行特征提取和模型构建，分析客户的信用状况，预测违约风险，帮助银行等金融机构决定是否发放贷款、确定贷款额度及利率等，例如分析一个中小企业的资金流水、上下游交易情况来评估其还款能力和信用等级

2.2 金融市场数据分析

证券交易所、投资银行等会处理海量的金融市场数据，如股票价格走势、成交量、宏观经济指标、行业数据等。Hadoop 可以存储这些时间序列数据，并结合数据分析工具对其进行相关性分析、趋势预测等，辅助投资者制定投资策略，例如通过分析过去多年不同行业股票价格与宏观经济因素的关联，判断在当前经济形势下哪些行业更具投资潜力

3.电信行业

3.1 客户关系管理

电信运营商拥有庞大的客户群体，积累了诸如用户通话记录、短信内容、上网流量使用情况、套餐订购历史等丰富的数据。借助 Hadoop 存储这些数据后，进行数据挖掘和分析，可以深入了解客户的使用习惯、需求偏好，进而实现精准营销，比如针对流量使用大户推荐更合适的大流量套餐，对通话时长较长的客户推荐优惠的通话套餐等，同时还能及时处理客户投诉，提升客户满意度

3.2 网络优化与运维

电信网络会产生大量的设备运行日志、网络性能指标数据等，通过 Hadoop 收集并分析这些数据，能够发现网络中的故障点、拥塞区域，评估网络设备的健康状况，为网络的优化升级、基站的合理布局、资源的有效调配等提供决策支持，确保网络的稳定和高效运行，例如根据不同区域基站的信号覆盖范围、用户接入数量等数据来决定是否需要增设基站或调整基站发射功率

4.零售行业

4.1 销售数据分析

零售商需要处理海量的销售记录，包括商品销售数量、销售额、库存变化、顾客购买时间、购买渠道等数据。利用 Hadoop 存储并分析这些数据，可以分析出不同商品的销售趋势、季节性波动、地区差异等，帮助商家制定采购计划、调整商品定价、优化店铺布局，例如大型连锁超市通过分析各门店不同季节商品的销售情况来提前备货、调配库存，避免积压或缺货现象

4.2 供应链管理

从供应商、物流、仓储到销售终端整个供应链环节都会产生大量的数据，Hadoop 可以存储这些数据并进行整合分析，实现对供应链的可视化管理，优化物流配送路径、预测需求、降低库存成本等，比如通过分析历史订单数据以及交通路况等信息来规划最优的物流配送路线，提高配送效率，减少运输损耗

5.医疗行业

5.1 医疗影像分析

医院会产生大量的 X 光、CT、MRI 等影像资料，这些影像数据量极大且需要长期保存。Hadoop 的 HDFS 可提供大容量存储解决方案，同时可以结合机器学习算法（在 Spark 等计算框架上运行）对影像进行特征提取、疾病诊断辅助，例如通过分析大量的肺部 CT 影像来辅助医生识别早期肺癌病变特征，提高诊断的准确性和效率

5.2 临床数据研究

收集患者的病历、症状、治疗方案、用药情况、基因数据等多方面临床数据，利用 Hadoop 进行存储和分析，有助于医学研究人员开展疾病的发病机制研究、药物疗效评估、个性化医疗方案制定等工作，比如通过分析大量糖尿病患者的用药和血糖控制数据来评估不同药物的治疗效果，为研发更有效的治疗方法提供参考

6.能源行业

6.1 电力系统监测与管理

电力公司要面对大量来自发电厂、变电站、输电线路等设备的运行数据，如发电量、电压、电流、设备温度等实时监测数据以及历史运维数据。Hadoop 可以存储这些海量数据，并通过分析来预测设备故障、优化发电计划、调配电力资源等，例如通过分析过往变压器故障前的各项运行参数变化来提前预警可能出现的故障，保障电力系统的安全稳定运行

6.2 油气勘探与生产

在油气行业，勘探阶段会有海量的地质数据，生产阶段也有油井产量、压力、含水率等数据。利用 Hadoop 存储和分析这些数据，能辅助确定油气资源的分布、优化油井开采方案、提高采收率等，比如通过分析不同区域的地质构造和油井历史生产数据来制定更合理的油井压裂、注水等增产措施

7.政府部门

7.1 智慧城市建设

涉及到交通、安防、环保、政务等多个领域的数据整合与分析。例如交通管理部门利用 Hadoop 存储和分析道路监控摄像头采集的车辆流量、车速等交通数据，实现智能交通调度，缓解拥堵；安防部门通过分析监控视频、报警记录等数据来提升城市治安防控水平；环保部门处理空气质量监测、水污染监测等数据来制定更有效的环境治理策略等，全面提升城市的智能化管理水平

7.2 人口普查与统计分析

每过一段时间进行的人口普查会产生海量的人口信息数据，包括个人基本信息、家庭情况、就业情况等。Hadoop 可以存储这些大规模数据，并进行数据挖掘和分析，辅助政府制定相关政策，比如分析不同地区的人口年龄结构、劳动力分布等来规划教育资源、就业培训、养老设施等方面的建设

互联网公司：

用户行为分析：互联网公司使用 Hadoop 来分析用户的浏览和点击行为，从而优化广告投放、推荐系统和内容个性化。
日志分析：Hadoop 被用于处理和分析大量的服务器日志数据，以监控系统性能、检测异常和改进用户体验。
金融行业：

风险管理：金融机构利用 Hadoop 分析大规模交易数据，评估风险，检测欺诈行为，并进行市场趋势分析。
实时数据处理：Hadoop 支持流数据处理，可以实时分析金融市场数据，做出快速决策。
医疗健康：

基因组学研究：Hadoop 被用于处理和分析大规模基因组数据，帮助科学家进行基因组学研究和疾病预测。
病历数据分析：医疗机构使用 Hadoop 分析患者的电子病历数据，提供个性化医疗服务和改进医疗质量。
电信行业：

网络监控：电信公司利用 Hadoop 处理和分析大量的网络数据，监控网络性能，优化网络配置，并提供更好的用户体验。
客户分析：Hadoop 帮助电信公司分析客户的使用行为和需求，从而进行精准的市场营销和客户服务

集群架构

  • 172.16.100.101 hadoop01 NameNode/DataNode/Nodemanager
  • 172.16.100.102 hadoop02 SecondaryNameNode/DataNode/Nodemanager
  • 172.16.100.103 hadoop03 ResourceManager/NodeManager/DataNode

1.配置集群SSH免密

2.配置集群Java环境

yum install -y java-1.8.0-openjdk

3.配置集群

tar -xzvf hadoop-3.2.4.tar.gz
mv hadoop-3.2.4 /opt/hadoop && mkdir -p /opt/hadoop/data
useradd hadoop && chown -R hadoop.hadoop /opt/hadoop

3.1 环境变量配置

vi /opt/hadoop/etc/hadoop/hadoop-env.sh 


export HDFS_NAMENODE_USER=hadoop
export HDFS_DATANODE_USER=hadoop
export HDFS_SECONDARYNAMENODE_USER=hadoop
export JAVA_HOME=/usr/lib/jvm/jre-1.8.0-openjdk
export YARN_RESOURCEMANAGER_USER=hadoop
export YARN_NODEMANAGER_USER=Hadoop

3.2 集群核心配置

vi /opt/hadoop/etc/hadoop/core-site.xml


<configuration>
    <!-- 设置NameNode地址 -->
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://hadoop01:8020</value>
    </property>

    <!-- 设置Hadoop数据存储目录 -->
    <property>
        <name>hadoop.tmp.dir</name>
        <value>/opt/hadoop/data</value>
    </property>

    <!--设置HDFS网页登录用户为 -->
    <property>
      <name>hadoop.http.staticuser.user</name>
      <value>hadoop</value>
    </property>
</configuration>

3.3 HDFS配置

vi /opt/hadoop/etc/hadoop/hdfs-site.xml


<configuration>
    <!-- 设置NameNode控制界面访问地址 -->
    <property>
      <name>dfs.namenode.http-address</name>
      <value>hadoop01:9870</value>
  </property>
    <!-- 设置SecondaryNameNode控制界面访问地址 -->
  <property>
      <name>dfs.namenode.secondary.http-address</name>
      <value>hadoop02:9868</value>
  </property>
</configuration>

3.4 YARN配置

vi /opt/hadoop/etc/hadoop/yarn-site.xml
